# 推送安全校验详细步骤

本文件是项目 `AGENTS.md` 与 `CLAUDE.md` 共用的推送安全详细参考；根文件中的安全红线优先。

## 校验命令

```bash
node scripts/pre-push-check.mjs           # 普通推送
node scripts/pre-push-check.mjs --release # 发布版本
```

## 校验项目

### 1. Git 状态与分支
- 当前分支是否正确
- 是否有未提交的更改
- 本地是否落后于远程

### 2. 敏感信息检查
- Token / Key / Secret / Password
- SSH 私钥
- 数据库连接串
- 内网地址 / 内部 IP

### 3. 敏感文件检查
- `.env` 文件
- 数据库文件（`.db` / `.sqlite`）
- 日志文件
- 缓存文件
- 备份文件

### 4. 路径与地址检查
- 本机绝对路径（如 `C:\Users\...`）
- 内网地址
- 本地开发 URL

### 5. 构建产物检查
- `dist/` 目录
- `target/` 目录（Rust）
- `node_modules/`
- 超大文件（>10MB）

### 6. 依赖与许可证
- 新增依赖的许可证是否合规
- 是否有已知安全漏洞

### 7. 代码质量
- lint 检查
- 类型检查
- 构建是否通过
- cargo check（Rust 部分）

## 输出格式

校验完成后输出：

- ✅ 通过项
- ⚠️ 警告项
- ❌ 阻断项
- 待推送的分支、提交和文件摘要
- 建议的下一步操作

## 阻断项处理

- 存在任何阻断项时，**即使用户要求继续，也不得推送**
- 必须先修复阻断项并重新校验通过

## 结尾声明

每次校验输出末尾**必须**明确写出：

> 当前未执行任何推送、打 tag 或 Release 操作，等待用户下一步明确指示。

## 确认触发词

仅在用户查看校验结果后，再次明确回复以下指令时才允许推送：

- "确认推送" / "按当前结果推送" / "确认打 tag" / "确认发布 Release"
